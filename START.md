# Локальный запуск проекта

Для корректной работы приложения локально необходимо подготовить окружение: актуализировать API-клиент и поднять сервис авторизации.


### Шаг 1. Генерация API-клиента
Перед первым запуском (или после любого изменения спецификации бэкенда в swagger.yaml) необходимо сгенерировать DTO-модели и HTTP-сервисы.

```bash
npm run api:gen
```



### Шаг 2. Запуск сервиса авторизации (Keycloak)
Авторизация в проекте завязана на Keycloak. Для локальной разработки он разворачивается через Docker. Убедитесь, что Docker-демон запущен, и поднимите контейнеры из docker-compose.yml:

```bash
docker compose up -d
```


После этого можно запустить Angular:

```bash
npm run start
```
